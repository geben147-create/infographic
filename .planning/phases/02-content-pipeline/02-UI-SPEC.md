# Phase 2 — UI Design Contract

> Visual and interaction contract for Phase 2: Content Pipeline.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> **Scope note:** Phase 2 has no browser frontend. All "UI" is:
> (a) FastAPI JSON response schemas — the operator's programmatic interface
> (b) `rich`-rendered CLI output — real-time pipeline progress and cost display
> (c) Field naming and copywriting conventions — status labels, error messages, log lines
>
> Browser dashboard is Phase 3 (OPS-06). Do not implement HTML views in Phase 2.

## Typography

Not applicable to browser rendering.

For `rich` CLI output, use the following text style conventions:

| Role | rich Style | Usage |
|------|-----------|-------|
| Body | default (white/light) | Regular log lines, info messages |
| Label | bold | Column headers, field names in status output |
| Heading | bold + color="bright_white" | Section headers (e.g., "Pipeline Started") |
| Accent | color="orange1" | Cost values, dollar amounts, progress percentages |
| Success | color="green" | Completion messages, upload URLs |
| Error | color="red" bold | Error messages, failure reasons |
| Muted | color="bright_black" | Secondary details, file paths, run IDs |

Color palette source: User preference — "beige, orange tones (reference claude.ai)". Orange1 (`#FF8700`) is the `rich` standard closest to claude.ai orange. Beige is approximated by `bright_white` on dark terminals.

## API Response Schema Contract

These Pydantic response models define the JSON contract for Phase 2 FastAPI endpoints. Field names must match exactly — they are the UI seen by the operator.

### POST /api/pipeline/trigger

```
PipelineTriggerRequest:
  topic: str              — required, the content topic keyword
  channel_id: str         — required, e.g. "channel_01"

PipelineTriggerResponse:
  workflow_id: str        — e.g. "pipeline-abc12345"
  status: "started"       — literal, always "started" on success
  channel_id: str         — echo of input
  topic: str              — echo of input
  estimated_cost_usd: float | None  — null if vgen_enabled=False, else "$0.20 x N scenes"
```

### GET /api/pipeline/status/{workflow_id}

```
PipelineStatusResponse:
  workflow_id: str
  status: PipelineStatus  — enum: "running" | "completed" | "failed" | "unknown"
  current_step: str | None  — e.g. "generate_script" | "image_gen_scene_03" | "youtube_upload"
  scenes_total: int | None
  scenes_done: int | None
  cost_so_far_usd: float | None  — live running total from cost_log.json
  youtube_url: str | None  — populated on completion
  error: str | None       — populated on failure
  started_at: str | None  — ISO 8601
  completed_at: str | None  — ISO 8601
```

### GET /api/pipeline/cost/{workflow_id}

```
CostDetailResponse:
  workflow_id: str
  channel_id: str
  total_cost_usd: float
  breakdown: list[CostLineItem]

CostLineItem:
  service: str     — "fal.ai" | "none"
  step: str        — e.g. "video_gen_scene_01"
  amount_usd: float
  resolution: str | None  — "480p" | "720p" | null
```

Source: REQUIREMENTS.md VGEN-02 (real-time cost display), PIPE-05 (upload status), derived from existing Phase 1 response pattern in src/api/sync.py

## Copywriting Contract

### API Status Values (PipelineStatus enum)

Use exactly these string values — not "RUNNING", not "in_progress":

| Status | Value | When Used |
|--------|-------|-----------|
| Running | `"running"` | Workflow executing, any step |
| Completed | `"completed"` | Final MP4 uploaded, cleanup done |
| Failed | `"failed"` | Any unrecoverable activity failure |
| Unknown | `"unknown"` | Workflow handle not found or Temporal error |

### current_step Labels

Use these exact strings for `current_step` field — executor must match these exactly:

| Step | Label |
|------|-------|
| Dir setup | `"setup_dirs"` |
| Script generation | `"generate_script"` |
| Image generation (scene N) | `"image_gen_scene_{N:02d}"` |
| TTS generation (scene N) | `"tts_scene_{N:02d}"` |
| Video clip (scene N) | `"video_clip_scene_{N:02d}"` |
| Thumbnail generation | `"generate_thumbnail"` |
| Video assembly | `"assemble_video"` |
| YouTube upload | `"youtube_upload"` |
| Cleanup | `"cleanup"` |

### CLI Output Copywriting (rich)

| Element | Copy |
|---------|------|
| Primary CTA (pipeline start) | `"Pipeline started"` + workflow_id |
| Progress header | `"Generating: {topic}"` |
| Scene progress | `"Scene {N}/{total}: {step_name}"` |
| Cost line | `"Cost so far: ${amount:.2f}"` |
| Completion | `"Upload complete: {youtube_url}"` |
| Empty state (no topic given) | `"No topic provided. Usage: POST /api/pipeline/trigger with topic and channel_id"` |
| Error state | `"Pipeline failed at {step}: {error_message}. Check logs or retry: GET /api/pipeline/status/{workflow_id}"` |
| fal.ai unavailable | `"Video gen disabled (FAL_KEY not set). Using Ken Burns fallback."` |
| Quota exceeded | `"YouTube quota exceeded for channel {channel_id}. Try again after {reset_time} UTC."` |

### Destructive Action Confirmations

Phase 2 has one destructive action: pipeline cancellation (Temporal workflow termination).

| Action | Trigger | Confirmation Approach |
|--------|---------|----------------------|
| Cancel pipeline | `DELETE /api/pipeline/{workflow_id}` | HTTP DELETE semantics — no additional confirmation dialog (no UI). Response must include `{"cancelled": true, "workflow_id": "..."}` |
| Cleanup intermediate files | Automatic post-completion | No confirmation needed — cleanup activity runs automatically, documented in D-14 |

Source: REQUIREMENTS.md PIPE-01 through PIPE-06, VGEN-02, user preferences for clean/minimal output

## Rich CLI Output Layout

The `rich` progress display for a running pipeline must follow this layout:

```
╭─────────────────────────────────────────────────────╮
│ Generating: "한국 부동산 투자 가이드"                 │
│ Channel: channel_01   Run: pipeline-abc12345         │
╰─────────────────────────────────────────────────────╯

 Progress  [████████░░░░░░░░░░░░] 40%  Scene 4/10
 Step      image_gen_scene_04
 Cost so far  $0.80

 Scene 01  script ✓  image ✓  tts ✓  video ✓
 Scene 02  script ✓  image ✓  tts ✓  video ✓
 Scene 03  script ✓  image ✓  tts ✓  video ✓
 Scene 04  script ✓  image ✓  tts ✓  video ⟳
```

Color mapping:
- Panel border: `bright_black` (muted, Notion-style)
- Scene checkmarks `✓`: `green`
- In-progress `⟳`: `orange1`
- "Cost so far" value: `orange1` bold
- Progress bar fill: `orange1`
- Percentage: `orange1`

Source: User preference "Figma, Notion style (clean, minimal, well-structured)" + "warm, professional dashboard aesthetic"
